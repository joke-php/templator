**При рендере localVars вероятно не нужны**
**TemplateEngine возвращает всегда HTML**
Работа с файлами и кешем должна быть внутри

**Реализация умного компонента**

1.  **Архитектура:** Единый скомпилированный PHP-файл шаблона для всех пользователей.
2.  **Логика `Static/Cache` хендлера:**
    *   Генерирует PHP-код проверки кэша (по ключу, зависящему от контекста, например, роли пользователя).
    *   При промахе кэша (cache miss) выполняет рендеринг блока «на лету» и сохраняет результат в файл/хранилище.
    *   При попадании (cache hit) выводит готовый HTML.
3.  **Данные:** Компонент сам запрашивает необходимые данные через DI-контейнер (сервисы), если они не переданы явно в `$context`.
4.  **Хранение:** Разделить кэш **кода** (скомпилированные `.php` файлы шаблонов) и кэш **данных** (HTML-фрагменты компонентов).
```php
<?php
$cacheKey = 'html_admin_menu';
if ($html = Cache::get($cacheKey)) {
    echo $html;
} else {
    ob_start();
    
    $items = \App\Services\MenuService::getAdminMenu(); 
    
    foreach ($items as $item) {
        echo '<li>' . htmlspecialchars($item['name']) . '</li>';
    }
    
    $html = ob_get_clean();
    Cache::put($cacheKey, $html, 3600);
    echo $html;
}
?>
```

**Добавление отслеживания позиции**
В TokenInterface или базовый класс Token добавить свойства $line и $column.
Лексер должен заполнять их при создании токенов.
Парсер и Компилятор должны передавать эту информацию в исключения.

**Реализация Raw-вывода**
Добавить новый тип токена/узла для "сырого" вывода (например, {!! $var !!} или {% raw %}).
Или добавить модификатор к существующему выводу.
Убедиться, что стандартный вывод ({{ }}) всегда экранирует данные для защиты от XSS.
